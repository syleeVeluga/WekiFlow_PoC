

# **🚀 \[Final PRD v4.0\] WekiFlow (위키플로우)**

**Hybrid RAG(Vector+Graph) & Sandboxed Terminal 기반 엔터프라이즈 지식 형상관리 워크스페이스**

## **1\. 💻 고도화된 통합 기술 스택 (인프라 확장)**

100% TypeScript 생태계 위에 **격리 런타임**과 **그래프 DB 계층**을 추가하여 에이전트의 능력을 극대화합니다.

| 계층 (Layer) | 주요 기술 (Tech Stack) | 핵심 역할 및 도입 목적 |
| :---- | :---- | :---- |
| **Frontend** | **Vite React \+ TS** | SPA. BlockNote(일반 열람) ↔ Monaco Diff(정밀 검토) 하이브리드 토글 UI. |
| **Agent Core** | **Vercel AI SDK \+ TS** | 듀얼 파이프라인 오케스트레이션. (메인 병합 워커 & 트리플 추출 서브 워커) |
| **Sandboxing** | **E2B (e2b.dev)** 또는 격리 Docker | **\[신규\]** 에이전트가 격리된 클라우드 환경에서 터미널(bash), python 코드를 안전하게 실행할 수 있는 일회성(Ephemeral) 런타임. |
| **Queue** | **Redis \+ BullMQ (TS)** | 대용량 백그라운드 워커 노드. (Main Task 큐 / Graph Extraction 큐 분리) |
| **Vector DB** | **MongoDB Vector Search** | 기존 청크(Chunk) 단위의 의미론적(Semantic) 임베딩 검색. |
| **Graph DB** | **Neo4j** (또는 MongoDB $graphLookup) | **\[신규\]** 추출된 트리플렛(Entity-Relation-Entity) Graph 지식망 통합 저장. |
| **Storage** | **MinIO (Docker)** | 원본 파일 및 마크다운/이미지 에셋 저장, 샌드박스 마운트용 볼륨. |

## **2\. 🔄 듀얼 에이전트 파이프라인 (The Hybrid Loop)**

기존의 단일 파이프라인을 \[메인 문서 병합 루프\]와 **\[지식 그래프 추출 루프\]** 두 개의 맞물리는 톱니바퀴로 완벽히 분리합니다.

### **🌊 \[파이프라인 A\] 메인 워커 에이전트 (작성/병합/검증)**

정보가 인입되면 가동됩니다. 에이전트는 Vector DB의 한계를 넘기 위해 클라우드 터미널을 도구(Tool)로 자유롭게 꺼내 씁니다.

Plaintext  
\[1. 지식 인입\] 텍스트/파일 업로드 또는 데이터 소스 수집 ➜ BullMQ \[Main Queue\] 적재  
       ↓  
\[2. 하이브리드 다차원 검색 (에이전트 도구 자율 선택)\]  
   ├─ ① tool\_search\_vector: "연차 휴가" 관련 의미론적 유사 문서 탐색.  
   ├─ ② tool\_search\_graph: \[파이프라인 B\]가 만든 지식망을 탐색하여 연관 부서/규정 다중 홉(Multi-hop) 추론.  
   └─ ③ tool\_execute\_sandbox\_terminal \[🌟핵심\]:   
        "에이전트 판단: Vector 검색만으로는 특정 규정 번호나 금액을 확신할 수 없어."  
        ➜ 클라우드 샌드박스 터미널을 띄우고 대상 MD 파일들에 대해  
          \`grep \-rnw "제4조 2항" ./docs\` 또는 \`python data\_parser.py\`를 직접 코딩/실행하여   
          단어 하나 놓치지 않는 100% 확정 탐색 및 read 수행.  
       ↓  
\[3. 병합 및 자가 검증\]  
   수집된 완벽한 팩트를 바탕으로 문서 병합(tool\_merge) 후 무결성 검증(tool\_verify\_integrity).  
       ↓  
\[4. 리뷰 및 배포\]  
   관리자가 '🔴 검토' 메뉴의 Monaco Diff 화면에서 코드 대조하듯 검수 후 \[✅ 승인(Commit)\].

### **🧬 \[파이프라인 B\] Triple 기반 지식 그래프 추출 에이전트 (Post-Process)**

관리자가 승인하여 문서가 🔷 조직 지식으로 배포되는 즉시, **백그라운드에서 두 번째 워커가 가동**됩니다.

Plaintext  
\[1. 트리거\] 메인 파이프라인에서 MD 파일이 승인됨 ➜ BullMQ \[Graph Queue\] 적재  
       ↓  
\[2. 트리플(Triplet) 추출 (LightRAG / SimGRAG 방법론 차용)\]  
   서브 에이전트가 배포된 MD 텍스트를 읽고 시스템 프롬프트 체인을 실행하여 트리플 노드 자동 파싱.  
   \* 프롬프트 지시: "너는 지식 추출기다. 문서를 분석하여 (Subject)-\[Predicate\]-\>(Object) 쌍의 JSON 배열로 추출하라. 모호한 대명사는 문맥을 파악해 원본 명사로 치환하라."  
   \* 추출 예시:   
     \- \`(신입사원) \-\[부여받는다\]-\> (연차 15일)\`  
     \- \`(연차 규정) \-\[결재 권한자\]-\> (부서장)\`  
       ↓  
\[3. 분류 및 Graph DB 적재\]  
   추출된 트리플 데이터를 Graph DB에 병합(Merge) 저장. 유사/동일 엔티티는 자동으로 병합(Entity Resolution).  
       ↓  
\[4. 메인 에이전트에 환원 (선순환)\]  
   이렇게 구축된 거대한 지식망(Knowledge Graph)은 다음 번 \[파이프라인 A\]가 실행될 때,  
   단순 키워드 매칭을 뛰어넘는 '다차원 관계망 쿼리'의 핵심 소스로 활용됩니다.

## **3\. 🤖 강화된 핵심 에이전트 도구 (Agent Tools 명세)**

Vercel AI SDK에 주입될 핵심 Tools(함수)들이 한 차원 진화합니다.

1. **tool\_execute\_sandbox\_terminal (클라우드 쉘/코딩 도구):**  
   * **기능:** E2B 환경 내에서 bash 명령어, python 스크립트를 실행하고 stdout/stderr 결과를 반환합니다.  
   * **기대효과:** AI 특유의 할루시네이션을 원천 차단합니다. 에이전트가 "내가 모르는 게 있으면 파일시스템을 find하고 grep하거나 cat해서 직접 읽어봐야겠다"라고 판단하여 Goose처럼 능동적으로 컴퓨터를 조작합니다.  
2. **tool\_search\_graph (지식 그래프 탐색기):**  
   * **기능:** "영업팀 신입사원이 갈 수 있는 출장 범위는?" 같이 여러 규정이 얽힌 질문에 대해, (영업팀)-(신입사원)-(출장규정) 트리플 관계선을 타고 추론하여 완벽한 컨텍스트를 가져옵니다.  
3. **tool\_extract\_triplets (트리플 추출기 \- 파이프라인 B 전용):**  
   * **기능:** 배포된 최신 MD 문서의 문맥을 쪼개어 정형화된 JSON 형태의 Triple 지식 그래프로 변환하고 DB에 적재합니다.

## **4\. 🎨 UI/UX 매핑 (기존 시안 유지 및 기능 심화)**

* **🔴 검토 (14)**: 파이프라인 A의 **병합 및 자가 검증**이 완료된 상태. 진입 시 **Monaco Diff**를 통해 관리자의 최종 판단(Human-in-the-loop) 진행.  
* **🔷 조직 지식 (88)**: 파이프라인 B의 **트리플 추출(GraphRAG)이 진행**되는 공식 문서 저장소. 진입 시 **BlockNote**로 렌더링.  
* **🔗 데이터 소스 / ✏️ 직접 추가**: 정보를 입력하여 파이프라인 A(Main Queue)를 트리거하는 시작점.  
* **📁 문서 트리**: 에이전트가 문서 구조를 파악하고 자동 생성/분류한 인접 리스트 기반 무한 뎁스 폴더.

## **5\. 🚀 단계별 액션 플랜 (수정된 로드맵)**

구조가 거대해진 만큼, 핵심 코어 모듈의 개별 PoC(개념 증명)가 최우선입니다.

### **🚩 Phase 1: 투트랙 에디터 UI 및 기본 인프라 뼈대**

* Vite React (BlockNote ↔ Monaco 하이브리드 토글 컴포넌트) 개발.  
* Node.js 백엔드 \+ BullMQ 메인 큐 시스템 세팅.

### **🚩 Phase 2: 샌드박스 터미널 통합 (파이프라인 A 코어)**

* **\[가장 중요\]** Vercel AI SDK에 E2B.dev (또는 격리 컨테이너) 연동.  
* 에이전트가 tool\_execute\_sandbox\_terminal을 통해 grep, awk 명령어로 로컬 문서 저장소에서 팩트를 정확히 빼오는지 집중 테스트.

### **🚩 Phase 3: Triple 트리플 추출 파이프라인 도입 (파이프라인 B 코어)**

* 문서 배포(Commit) 이벤트 발생 시 동작하는 백그라운드 Graph 워커 구성.  
* LightRAG 시스템 프롬프트를 차용해, 복잡한 텍스트에서 Entity, Relationship을 JSON 배열로 추출 및 Graph DB에 일관되게 적재하는 로직 구현.

### **🚩 Phase 4: 궁극의 하이브리드 RAG 통합 완성**

* 구축된 지식 그래프를 파이프라인 A의 탐색 도구(tool\_search\_graph)로 연결 및 시스템 전면 통합.

### **💡 실행 제안 (Action Item)**

고객님의 통찰력 덕분에 WekiFlow는 **"AI가 헷갈리면 스스로 리눅스 터미널을 열어 팩트체크를 하고, 작업이 끝나면 뒷단에서 지식망 노드를 이어서 다음 작업을 똑똑하게 준비하는"** 최상위 레벨의 지능형 시스템이 되었습니다.  
본격적인 개발을 착수하신다면 메인 비즈니스 로직에 앞서 아래 두 가지 코어 스크립트를 먼저 테스트해 보시는 것을 강력히 권장합니다.

1. **\[E2B 샌드박스 연결 테스트\]:** 에이전트가 터미널 도구를 호출해 직접 grep을 실행하는 TypeScript 코드.  
2. **\[Triple 추출 프롬프트 테스트\]:** 사내 규정 텍스트를 넣었을 때 유효한 형태의 \[Entity \- Relation \- Entity\] JSON을 뱉어내는지 확인하는 파이프라인.
